# 1층 베이스 — 자연스러움 복원 규칙

모든 톤 변환에 **공통으로 먼저** 깔리는 바닥 규칙이다. 어떤 말투를 입히든, 그 전에 "사람이 쓴 글의 결"을 복원한다. 2026년 기준 실제 글(velog·브런치·링크드인·Medium)을 시기별로 비교해 뽑은 마커이며, 근거는 [research-notes.md](research-notes.md)에 있다.

핵심 통찰 하나만 먼저: **AI 티는 "최신 글이라서"가 아니라 "정보·홍보 목적으로 매끈하게 쓰려다" 생긴다.** 그래서 베이스 규칙의 목표는 "매끈함을 일부러 흐트러뜨려" 사람 손맛을 되살리는 것이다.

---

## B1. 리듬을 일부러 들쭉날쭉하게 (burstiness)

AI 글의 1번 신호는 문장 길이가 고른 것이다. 사람은 긴 설명 뒤에 짧게 못을 박는다.

- 한 문단 안에 **긴 문장 + 3~6음절 단언/감탄**을 섞는다.
  - 나쁨: "이번 분기에는 여러 지표가 개선되었고 팀의 협업도 원활했으며 결과적으로 목표를 달성했습니다."
  - 좋음: "이번 분기엔 지표가 거의 다 올랐다. 협업도 매끄러웠고. 목표? 넘겼다."
- 모든 문장을 비슷한 중간 길이로 만들지 않는다. 가끔 한 문장은 일부러 짧게 끊는다.

## B2. 종결어미를 다양하게 (같은 어미 3연속 금지)

- `-습니다`·`-요`·`-다`를 3문장 연속 쓰지 않는다.
- 명사형 종결("~인 셈."), 의문형("아닐까?"), 말끝 흐림("...였달까", "~더라")을 섞는다.
- 단, **톤마다 허용 범위가 다르다.** 격식체(하십시오체)에서는 말끝 흐림을 자제한다. 톤 레이어가 이 범위를 정한다 → [tone-presets.md](tone-presets.md).

## B3. 문두 접속사를 절반 이상 덜어낸다

한국어는 접속사 없이 두 문장을 나란히 두면 인과·대조가 저절로 읽힌다.

- "또한 / 그리고 / 하지만 / 따라서 / 게다가 / 나아가"를 문단 첫머리에 기계적으로 박지 않는다.
- 지웠을 때 뜻이 통하면 지운다. 특히 "결국 / 따라서 / 요약하면" 같은 결산 피벗은 1~2회로 제한.

## B4. 1인칭 + 검증 가능한 구체로 (추상어 추방)

화자가 사라지고 일반론만 남으면 AI 티가 난다.

- 무생물·추상 주어 + 만능 동사를 사람 주어 + 동사로.
  - "이 보고서는 인사이트를 제공합니다" → "이 글 보면 ~를 알 수 있어요"
  - "시장은 변화를 겪을 예정이며" → "시장이 곧 바뀐다"
- 추상명사("효율성·중요성·가능성")로 끝나는 문장을 숫자·고유명사·도구명으로 교체.
  - "효과적이었다" → "10분 걸리던 게 2분으로 줄었다"
  - "많은 사용자가" → "내 주변 세 명이"

## B5. 번역투·명사화를 푼다

| 번역투 | 자연스럽게 |
|---|---|
| ~에 대해(서) | 목적격 조사로 직결 ("X에 대해 논의" → "X를 논의") |
| ~를 통해 (남발) | "~로", "~해서", "~함으로써"로 분산 |
| ~에 있어(서) | "~에서", "~을 볼 때" |
| ~을 가지고 있다 / have·make 직역 | 형용사·동사로 ("경쟁력을 가지다" → "경쟁력이 있다") |
| ~되어진다 (이중 피동) | "~된다" |
| ~에 의해 (피동) | 행위자를 주어로 ("AI에 의해 생성" → "AI가 만든") |
| ~의 분석을 진행했다 (명사화) | 동사로 ("분석해 봤다") |
| 단순한 X가 아니라 Y다 | 대조 클리셰. 한 번만 쓰거나 평서문으로 |

## B6. 도입은 사적으로, 마무리는 여운으로

- 거대담론 도입 금지: "오늘날 급변하는 시대에", "~은 점점 더 중요해지고 있습니다", "안녕하세요! 요즘 ~하실 텐데요" → 구체 장면·사건·날짜로 연다.
- 교훈조 마무리 금지: "결론적으로 ~가 중요합니다", "~란 결국 ~이다", "선택이 아닌 필수입니다" → 사적 다짐·질문·장면으로 닫는다.

## B7. 포맷 장식은 톤이 정한다 (일괄 규칙 아님)

이게 중요하다. **"이모지·불릿을 줄여라"를 모든 글에 똑같이 적용하면 안 된다.**

- 링크드인: 이모지·한 줄 단락·해시태그는 AI 티가 아니라 플랫폼 관습 → 살린다.
- 브런치 에세이: 볼드 소제목·번호목록·CTA가 늘면 AI/블로그화 신호 → 덜어낸다.
- velog: 코드블록·스크린샷은 자연스러움. 단 과도하게 대칭인 1·2·3 구조는 푼다.

→ 구체 분기는 [tone-presets.md](tone-presets.md)의 각 프리셋 "포맷" 항목을 따른다.

## B8. 구어 침범을 허용한다 (격식 톤 제외)

사람 글의 표식은 격식 안에 살짝 새어 드는 구어다.

- "~달까", "...", "ㅋㅋ", "ㄷㄷ", "허허", "~더라", "솔직히" 같은 입말이 **한두 번** 자연스럽게 끼어들도록 둔다.
- 단 빈도는 톤이 정한다. 하십시오체·비즈니스 격식에서는 0, 반말 친근·SNS·velog에서는 적극.

## 피해야 할 AI 티 신호 세트 (겹칠수록 강함)

변환 후 이 중 여러 개가 남아 있으면 다시 손본다.

1. 무생물 주어 + 동작 동사 ("보고서가 제공한다")
2. 종결어미·문장 길이 균일
3. 명사화 다발 (-성/-적/-화/-tion 누적)
4. 접속사 직역 남발 ("뿐만 아니라"=Moreover, "따라서"=Therefore)
5. 거대담론 도입 + 일반론/교훈 마무리
6. "X가 아니라 Y다" 대조 반복
7. em-dash(—)·세미콜론(;) 잔재 (영문 직역 흔적)
8. 정확히 3~4개로 떨어지는 대칭 병렬 불릿
9. hype 어휘("압도적·획기적·강력한") + "시사하는 바가 크다" 류

## 자가검증 (변환 직후 5초 점검)

1. **의미 100% 보존** — 사실·수치·고유명사·인용이 원문과 한 글자도 다르지 않은가
2. **위 AI 티 신호 세트** 중 남은 것이 2개 이하인가
3. **장르 이탈 없음** — 베이스를 적용하다 원문 장르를 망가뜨리지 않았는가
4. **톤 일치** — 요청한 말투의 register가 일관되는가 (다음 단계 tone-presets 점검과 연결)

위반 시 해당 부분만 롤백 후 재변환. 자체 루프 최대 1회.
