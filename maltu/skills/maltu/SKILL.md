# maltu — 한국어 말투 변환기

내가 준 한글 글을, 원하는 말투로 사람이 쓴 것처럼 자연스럽게 바꾼다.

## im-not-ai와의 경계 (헛수고 방지)

- **im-not-ai** (`/humanize`): AI 티 탐지·제거가 목적. 다단계 파이프라인, 정밀하지만 무겁다.
- **maltu**: "원하는 말투 입히기"가 목적. 가볍게 한 번에. 자연스러움 복원은 바닥에 깔고, 그 위에 톤을 얹는다.

탐지 리포트·정밀 감사가 필요하면 im-not-ai를 권한다. 여기서는 빠르게 톤을 바꾼다.

## 절대 원칙

의미·사실·주장·수치·고유명사·인용·인과·순서는 **한 글자도 바꾸지 않는다.** 바꾸는 건 어미·호칭·문장 리듬·표현·포맷뿐이다. 변경률이 과해지면(체감 30%↑) 내용을 건드린 것은 아닌지 점검한다.

## 2층 변환 모델

1. **1층 베이스 — 자연스러움 복원** (모든 톤 공통): [references/naturalness-base.md](references/naturalness-base.md)
   리듬 불균일·종결어미 다양화·접속사 절제·번역투 제거·구체 디테일 등 "사람 손맛"을 먼저 깐다.
2. **2층 톤 레이어 — 말투 적용**: [references/tone-presets.md](references/tone-presets.md)
   그 위에 지정한 말투를 입힌다. 톤이 베이스와 충돌하면 톤이 이긴다(예: 격식체는 구어 침범 0).

데이터 근거가 궁금하면: [references/research-notes.md](references/research-notes.md)

## 절차

1. **입력 확보**
   - 인자가 비었으면: "바꿀 글을 붙여넣어 주세요. 원하는 말투도 알려주시면 좋아요(예: 비즈니스 톤, velog 느낌)" 안내 후 종료.
   - 인자가 파일 경로(.txt/.md 등)면 Read로 본문을 불러온다.
   - 텍스트면 그대로 입력으로 쓴다.

2. **장르·플랫폼·동기 파악** (research-notes 핵심 발견 1)
   글이 회고/에세이/기술/홍보/공지 중 무엇이고 어디에 쓸 글인지 빠르게 잡는다. 이게 베이스 적용 방향을 정한다.

3. **톤 결정**
   - 사용자가 톤을 줬으면 그대로.
   - 안 줬으면 tone-presets의 프리셋을 짧게 제시하고 고르게 한다. (해요체 / 하십시오체 / 반말 친근 / SNS·톡 / 다정 상담 / 간결 드라이 / velog / 브런치 / 링크드인)
   - 프리셋에 없는 자연어 요청("교수님께 메일처럼")은 가장 가까운 프리셋을 베이스로 잡고 조정.

4. **변환**: 1층 베이스 → 2층 톤 순으로 적용. 포맷 규칙은 반드시 선택한 톤의 "포맷" 항목을 따른다(일괄로 이모지/불릿을 지우지 않는다).

5. **자가검증**: naturalness-base의 "자가검증" + tone-presets의 "톤 적용 후 점검"을 돌린다. 의미 훼손·register 흔들림·AI 티 잔존 시 해당 부분만 롤백 후 재변환(최대 1회).

6. **출력** (아래 형식).

## 출력 형식

```
[변환된 본문 — 마크다운 블록]
```

기본은 변환본만 깔끔히. 사용자가 원하거나 변경이 큰 경우 아래를 덧붙인다:
- **적용한 톤**: (예: velog 톤) — 한 줄
- **주요 변경 3건**: before → after (의미 보존을 보여주는 용도)

## 옵션 (인자 끝에 자연어로)

- `톤: <프리셋명 또는 자유 지시>` — 말투 지정
- `강도: 약하게 | 기본 | 적극` — 변환 정도 (기본값: 기본)
- `이 문단만` / `제목만` 등 — 범위 한정
- `요약 보여줘` — 주요 변경 before/after 표시

## 후속 작업

- "다른 톤으로 다시" → 같은 원문에 다른 프리셋 적용.
- "더 부드럽게/딱딱하게" → 같은 톤 안에서 구어 허용도·종결어미 조정.
- "이 부분만" → 지정 범위만 재변환.
