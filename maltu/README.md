# maltu — 한국어 말투 변환기

내가 준 한글 글을 **원하는 말투(톤)로, 사람이 쓴 것처럼 자연스럽게** 바꿔주는 Claude Code 플러그인.

의미·사실·수치·고유명사·인용은 그대로 두고 **어미·호칭·문장 리듬·표현·포맷만** 바꾼다.

## 무엇이 다른가

같은 마켓플레이스의 `im-not-ai`가 "AI 티 탐지·제거"에 초점을 둔 정밀·다단계 윤문이라면, `maltu`는 **"원하는 말투 입히기"**가 목적인 가볍고 빠른 변환이다.

- AI 티/번역투 제거가 주목적 → `im-not-ai`의 `/humanize`
- 원하는 톤으로 빠르게 변환 → `maltu`

## 2층 변환 모델

1. **1층 — 자연스러움 복원** (모든 톤 공통): 리듬 불균일, 종결어미 다양화, 문두 접속사 절제, 번역투·무생물 주어 제거, 추상어→구체 디테일.
2. **2층 — 톤 적용**: 그 위에 말투를 입힌다.

## 톤 프리셋

**일반 말투**: 해요체 · 하십시오체 · 반말 친근 · SNS·톡 · 다정 상담 · 간결 드라이

**플랫폼-네이티브** (실제 시기별 글 데이터로 검증): velog · 브런치 에세이 · 링크드인

프리셋에 없는 말투도 자연어로 요청 가능 — "교수님께 보내는 메일처럼", "사장님 보고 톤으로".

> 플랫폼마다 "AI 티"의 정의가 다르다. 예컨대 이모지·한 줄 단락은 링크드인에선 관습이지만 브런치 에세이에선 어색하다. 프리셋이 이 분기를 담는다.

## 사용법

```
/maltu 이 글 비즈니스 톤으로 바꿔줘: <붙여넣기>
/maltu ./draft.md velog 느낌으로
```

또는 그냥 자연어로 — "이 글 좀 더 친근하게 바꿔줘", "딱딱한데 부드럽게", "링크드인용으로".

## 데이터 근거

톤 규칙은 추측이 아니라 2023 이전 vs 2026 한국어 글(Medium·브런치·링크드인·velog) 비교에서 도출했다. 자세한 내용은 [`skills/maltu/references/research-notes.md`](skills/maltu/references/research-notes.md).

핵심 발견:
1. AI 티는 "시기"가 아니라 "장르·동기"가 가른다 — 개인 회고·에세이는 2026년에도 인간적.
2. 문체 전염 — AI를 비판하는 사람도 AI식 포맷을 흡수했다.
3. "AI 티"의 정의는 플랫폼마다 다르다 — 일괄 규칙은 위험.
